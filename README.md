# Foundation-of-Computer-science-Task-3

## General Overview
```
This GitHub project is a simple database design task based on a college that manages **student club memberships** (Music Club, Sports Club, Coding Club, etc.).

At the start, the college stores everything in **one table**, which causes repeated data and data errors. In this project, the data is fixed by doing **normalization (1NF, 2NF, 3NF)**, then creating an **ER Diagram**, and writing **SQL queries** (insert, display, and JOIN).
```

## Learning Objectives
```
By completing this project, I learned how to:

- Find problems in a single large table (duplicate data and redundancy)
- Explain why the table is **not normalized**
- Apply normalization steps:
  - **1NF**: keep values single (atomic) and choose a primary key
  - **2NF**: split data into Student, Club, and Membership tables
  - **3NF**: remove transitive dependencies
```
